# Estimador CAG

FastAPI service that turns a meeting transcription into a structured software estimate using **Context-Augmented Generation (CAG)**: static few-shot examples live in `app/context/examples.py` and are injected into the system prompt; the live meeting text is sent as the user message.

## Requirements

- Python 3.11
- [uv](https://docs.astral.sh/uv/)

## Setup

```bash
cd proyectos/estimador-cag
uv sync --group dev
cp .env.example .env
# Set OPENAI_API_KEY in .env (never commit .env).
```

## Run the API

```bash
cd proyectos/estimador-cag
uv run uvicorn app.main:app --reload
```

- Root: `GET http://127.0.0.1:8000/` (JSON pointers to docs and routes)
- Health: `GET http://127.0.0.1:8000/health`
- OpenAPI: `http://127.0.0.1:8000/docs`

Browsers may request `/favicon.ico`; there is no favicon asset, so that request may return 404 and can be ignored.

## Example request

```bash
curl -s -X POST http://127.0.0.1:8000/api/v1/estimate \
  -H "Content-Type: application/json" \
  -d '{"transcription":"The client needs a REST API for orders with idempotent POST."}'
```

Response fields include:

- `estimation`, `model`, `provider`
- `request_id`, `timestamp`, `latency_ms`
- `prompt_version`, `examples_version`
- `usage` (`prompt_tokens`, `completion_tokens`, `total_tokens`, `estimated_cost_usd`) only when `DEV_MODE=true`

### Response metadata (detailed)

The endpoint returns two kinds of metadata:

1. **Operational metadata** (always returned)
2. **Development metadata** (returned only when `DEV_MODE=true`)

#### Operational metadata (always)

- `request_id`: unique request identifier generated by the API (useful to correlate logs and client incidents).
- `timestamp`: UTC ISO-8601 timestamp indicating when the response was produced.
- `latency_ms`: total API processing time (milliseconds) for the estimation request.
- `prompt_version`: version identifier for the system prompt template currently in use.
- `examples_version`: version identifier for the static CAG examples currently injected.

`prompt_version` and `examples_version` are critical for reproducibility. If output quality changes over time, these fields let you trace whether behavior changed because of prompt edits, example edits, model updates, or infrastructure conditions.

#### Development metadata (`DEV_MODE=true`)

- `usage`: token usage reported by the model backend:
  - `prompt_tokens`
  - `completion_tokens`
  - `total_tokens`
  - `estimated_cost_usd` (approximate request cost, computed from token usage and model pricing known by the service)

When `DEV_MODE=false` (default), these fields are omitted from the response payload to keep production responses minimal and avoid exposing cost/usage internals by default.

#### Cost estimation notes

- Cost is an approximation (not billing source of truth).
- Cost appears only if:
  - `DEV_MODE=true`
  - provider returns token usage
  - model pricing is configured in the API
- If any of the above is missing, `usage.estimated_cost_usd` is `null`.

#### Versioning guidelines

Update metadata versions whenever behavior-affecting inputs change:

- Bump `prompt_version` when prompt instructions/format/constraints change.
- Bump `examples_version` when `app/context/examples.py` content changes.

Suggested convention:

- `v1`, `v2` for major behavioral changes
- `v1.1`, `v1.2` for incremental tuning

## Tests

```bash
cd proyectos/estimador-cag
uv run pytest
```

No real OpenAI calls: the suite mocks the provider client.

## Configuration

See `.env.example` for variable names. Required for live estimates:

- `OPENAI_API_KEY`

Optional overrides include `OPENAI_MODEL` (default `gpt-4o-mini`), `OPENAI_TIMEOUT_SECONDS`, and `DEV_MODE` (set `true` to include usage/cost metadata in API responses).
