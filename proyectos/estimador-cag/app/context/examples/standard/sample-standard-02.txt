# Estimation: Mobile Coffee Catalog Application MVP

---

## Assumptions

- The application will support both Android and iOS platforms with a shared codebase (using a framework like React Native or Flutter) to minimize development effort.
- Users will authenticate via email/password or social login.
- Coffee-related content (descriptions, images) is assumed to be provided and stable.
- The application will include basic user profiles, search functionality, and a follow feature.

---

## Architecture overview

### Frontend

- A shared mobile codebase utilizing React Native or Flutter for Android and iOS compatibility.

### Backend

- REST API to manage user data, coffee catalog, and follow relationships.
- User authentication and authorization management.

### Infrastructure

- Cloud hosting for the backend (e.g., AWS, Firebase).
- Database service for user and coffee catalog information (e.g., PostgreSQL, Firestore).
