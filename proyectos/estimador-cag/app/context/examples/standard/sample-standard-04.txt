## Estimación: Aplicación móvil de cafeterías (MVP)

### Assumptions

- La aplicación será desarrollada utilizando React Native o Flutter como tecnología cross-platform.
- La API REST se implementará utilizando una arquitectura estándar con autenticación JWT.
- El diseño UI/UX será limpio y funcional, sin personalización compleja.
- La escalabilidad se planificará para un uso moderado, acorde a un MVP.
- La carga de contenido inicial del catálogo es manual y no se prevén integraciones externas.

---

### Architecture Overview

- **Frontend:** Aplicación móvil realizada en React Native o Flutter, con componentes reutilizables y navegación sencilla.
- **Backend:** API REST para la gestión de usuarios, cafés y seguimientos. Implementación de endpoints seguros y protección de datos.
- **Infrastructure:** Base de datos para almacenamiento (por ejemplo, PostgreSQL o Firebase), autenticación a través de JWT, y despliegue en un entorno de nube (Heroku, AWS, etc.).

### Delivery Plan

- **Sprint count:** Aproximadamente 4 sprints
- **Sprint duration:** 2 semanas cada uno
- **Key milestones:**
  - Sprint 1: Autenticación, perfiles de usuario y gestión de usuarios.

### Risks

- El tiempo de implementación puede aumentar si se detectan necesidades de personalización adicional en la UI durante el desarrollo.
- La integración de login social puede presentar desafíos técnicos imprevistos.
- Los cambios en requisitos de negocio pueden impactar los tiempos de desarrollo.

Esta estimación proporciona un marco práctico y realista para el desarrollo de la aplicación móvil de cafeterías en su versión MVP, considerando las funcionalidades básicas y el enfoque en un lanzamiento rápido y efectivo.
