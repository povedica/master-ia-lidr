## Estimation: Mobile Habit Management App MVP (iOS & Android)

### Assumptions
- The app will support both iOS and Android platforms but will be built in parallel, sharing code where feasible.
- The design will prioritize functionality over aesthetics due to the "sencilla" interface requirement.
- The user registration process will rely on email and password authentication.
- Notifications will be handled via built-in push notification services for both platforms.
- Basic statistics will involve simple user inputs without complex analytics or graphs.

### Architecture Overview
- **Frontend**: Simple user interface for habit creation, registration, and basic statistics visualizations; developed using a cross-platform framework (e.g., React Native).
- **Backend**: REST API for user management, habit data storage, and push notification triggering; uses a database for persistence.
- **Infrastructure**: Deployment on a cloud service with basic CI/CD setup for automated builds and deployments.

### Delivery Plan
- Sprint count: 3
- Sprint duration: 2 weeks each
- Key milestones:
  - Sprint 1: Complete backend user management and habit APIs

### Risks
- Complexity in handling push notifications can lead to delays if platform-specific issues arise.
- User onboarding and habit setting complexity may require additional iteration based on user feedback.
- Unforeseen integration issues between shared code for iOS and Android may arise due to platform differences.
